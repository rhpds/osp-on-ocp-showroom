=== Setup validation

Participants should verify environment access with these steps:

**Verify OpenStack CLI access**:
In the bastion tab, run the following commands:
[source,bash]
----
# Once logged into bastion
oc project openstack
alias openstack="oc exec -t openstackclient -- openstack"
openstack server list
----

**Verify AAP project synchronization**:
Access {aap_controller_web_url}[AAP Console^] and navigate to:

* Automation Execution → Projects
* Locate "vmware migration toolkit project"
* Verify status shows "Successful" with green checkmark

== Troubleshooting guide

=== Common setup issues

**Problem**: "Permission denied" when attempting SSH to bastion

**Solution**:

. Verify you're using the correct port: `{bastion_ssh_port}`
. Confirm username is: `{bastion_ssh_user_name}`
. Check SSH command format: `ssh {bastion_ssh_user_name}@{bastion_public_hostname} -p {bastion_ssh_port}`
. If still failing, try adding `-v` flag for verbose output to diagnose connection issues

**Problem**: "Error: You must be logged in to the server (Unauthorized)" when running `oc` commands

**Solution**:

. Login to OpenShift from the bastion:
+
[source,bash,subs="attributes"]
----
oc login -u {openshift_cluster_admin_username} -p {openshift_cluster_admin_password} {openshift_api_url}
----
. Verify login was successful:
+
[source,bash]
----
oc whoami
oc project
----

**Problem**: "Cannot connect to AAP web interface" or browser shows certificate warning

**Solution**:

. Verify URL is correct: {aap_controller_web_url}
. Accept browser security exception for self-signed certificate if prompted
. Clear browser cache and cookies, then retry
. Try accessing from an incognito/private browser window

**Problem**: "openstack server list" returns empty or shows authentication error

**Solution**:

. Verify you're in the correct project:
+
[source,bash]
----
oc project openstack
----
. Confirm the openstackclient pod is running:
+
[source,bash]
----
oc get pods -n openstack | grep openstackclient
----
. If pod is not ready, wait 1-2 minutes and retry
. Re-create the alias:
+
[source,bash]
----
alias openstack="oc exec -t openstackclient -- openstack"
----

== Environment architecture

Your lab environment includes these pre-deployed components:

**Source environment**:

* VMware vCenter {vcenter_console} with test virtual machines
* Sample workloads configured with appropriate networking and storage

**Target environment**:

* Red Hat OpenShift cluster running OpenStack services
* OpenStack networks (public, private), security groups, and flavors pre-configured
* Conversion Host deployed in OpenStack for efficient VM data transfer

**Automation platform**:

* Ansible Automation Platform with migration execution environment
* Pre-configured credentials, inventories, and project repositories
* Migration playbooks from Red Hat OpenStack VMware Migration Toolkit

**Access infrastructure**:

* Bastion host with OpenStack CLI tools and SSH keys
* Pre-configured networking for access to all components
* OpenShift and AAP web consoles available via HTTPS
